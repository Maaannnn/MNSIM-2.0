摘要

当前结果说明什么
  • 当前主数据集共 72 个点，总体可行率约 79.2%，已经从“摸底”进入“收束验证”阶段。
  • 矩阵 A 已证明：P0/P1/P2 仍稳定可行，而 P3 在无补偿条件下整体失效，器件退化确实会触发设计迁移问题。
  • 矩阵 B 已证明：对 P1/P2，接口补偿保持 100% 可行，同时把最小时延改善到 3,884,383.15 ns、最小能耗压到 1,248,300.86 nJ。
  • 相对矩阵 A，当前最佳补偿点的时延下降约 545,641.84 ns，能耗下降约 1,944,759.29 nJ，说明补偿收益主要体现在 PPA 而不是精度提升。
  • 矩阵 D 已给出关键结论：P3 已被部分救回，当前恢复点是 `xbar=512x512 / ADC=7 / DAC=32 / PE=2x2`，精度 0.884766。
  • P4 目前仍是 0/4 可行，说明当前方案的有效边界大致停在 P3 左右，P4 更适合作为失效边界而不是主应用场景。
  • 矩阵 C 已证明：系统层 16/16 全可行，平均精度约 0.926514，说明当前器件/接口主方案在系统层也站得住。
  • `pe_num=2x2` 明显优于 `4x4`，因此论文默认并行度应固定在 `2x2`。
  • `tile_connection=3` 当前更稳，建议作为正式搜索默认拓扑。
  • `inter_tile_bw=80` 在当前样本下时延更优，建议保留为系统默认带宽。
  • 结构层面最稳定的结论仍是：`xbar_size=512x512` 应继续保留为主论文默认阵列尺寸。
  • 并行度上 `pe_num=4x4` 更稳，当前不建议回到更大的 `4x4` 作为默认设置。
  • 接口层当前可收束为两条论文候选线：`ADC=6 + DAC=128` 偏 PPA，`ADC=4 + DAC=32` 偏面积/基线。
  • 从已采样结果看，当前最稳的器件场景仍是 `P1`，但论文重点已经不再是“哪个 preset 最好”，而是“补偿能把退化器件拉回多少”。
建议下一步实验
  • 下一步不要再扩矩阵，直接基于当前结论构造正式论文版搜索空间并启动 NSGA-II / MOBO。
  • 正式搜索建议固定：`xbar_size=512x512`、`pe_num=2x2`、`tile_connection=2`、`inter_tile_bw=80`。
  • 正式搜索只保留少量关键变量：`rram_preset`、`adc_choice`、`dac_num`、`sub_position`，并围绕 P1/P2/P3 做论文主图。
风险提示
  • 当前 `analysis_scope` 仍以矩阵 A 为主，因为 A 的样本数最多；首页摘要已按全数据最新阶段重写，但下方部分图表仍更多体现 A 的统计。
  • 矩阵 B 目前只覆盖 P1/P2，所以 A/B 补偿报告不能被解释成“对全部 preset 都成立”的结论。
  • 矩阵 D 只出现了少量 P3 恢复点，现阶段更适合写成“边界恢复证据”，还不足以直接宣称对重度退化全面有效。
  • 矩阵 C 只覆盖 P1/P2，因此系统层默认结构是建立在中度退化场景上的；若后续要做 P3 系统级部署，还需补做更差器件下的系统验证。

怎么读这页
STEP 1
先看变量效应量排序
先判断优先调哪些变量。排名越靠前，说明这个变量改动后整体结果变化越大。
STEP 2
再看分组统计摘要
判断某个变量的哪个取值更像优选方向。这一步用于缩空间，不直接决定最终设计点。
STEP 3
再看器件迁移表
比较不同器件预设下，最优具体配置是否发生迁移。这一块最接近论文主结论。
STEP 4
最后看推荐配置
这里每一行都是完整配置点，才是最后能拿去做设计建议或论文表格的对象。
配置级最优
`推荐配置` 与 `按器件预设选出的当前最优平衡配置` 的最优对象，都是单个具体设计点（一个完整配置）。
分组级摘要
`变量影响摘要` 与 `变量效应量排序` 是按变量取值聚合后的统计，不代表该组内所有配置都优，也不等同于“最优设计”。
preset 点击
页面里出现的 `P0/P1/P2/...` 都可以点击，右侧抽屉会显示器件预设的具体阻值、波动与 SAF 参数。
