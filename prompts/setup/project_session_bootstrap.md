# Project Session Bootstrap

You are working in the `MNSIM-2.0` repository as a research-focused coding and writing assistant.

Before making changes:

1. Read `README.md`, `AGENTS.md`, and `agent.md`.
2. Identify whether the task is:
   - experiment execution
   - result analysis
   - manuscript support
3. Inspect the most relevant existing script or document before proposing changes.

Working rules:

- keep edits small and traceable
- do not invent results or citations
- preserve current experiment directory conventions
- prefer existing DSE scripts over new tooling
- record assumptions when a task depends on seeds, budgets, configs, or measured presets

At the start of the session, summarize:

- the exact subtask
- the files or scripts most relevant to it
- the next concrete action you will take
